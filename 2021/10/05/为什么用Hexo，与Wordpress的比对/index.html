为什么开始用Hexo，与Wordpress的比对

本文最后更新于：几秒前

趁着假期将域名从Godaddy搬回了国内，因为Godday上的域名续期比国内贵了一倍，反正都有备案号了；同时将香港主机的数据打包备份恢复到本地了，这个主机这几年基本就属于闲置白缴费的状态；为了督促自己多读书，准备重拾博客，重点是将开始尝试Hexo之旅，为什么放弃玩了十几年的Wordpress呢？

麦咭哥之所以放弃Wordpress，原因种种，跟Hexo相比大致列举如下：

Wordpress Hexo 结论 胜出
开发语言 PHP Node.js 除了Wordpress已经没有场合再使用PHP，而Node.js正普遍用于各项目 Hexo
数据库 MySQL 不需要 部署环境至简，能把数据库省掉自然是极好的 Hexo
运行环境 Apache/PHP/MySQL 纯静态页面 Hexo在本地直接生成了全部静态网页，这样发布到OSS都可以跑起来了 Hexo
后台管理 非常专业的管理界面 默认无管理界面，命令行 + Markdown编辑器 习惯了VSCode和npm命令行的同学后者不是难事，但对于其他行业的同学有门槛 Wordpress
网络安全 全球最大的博客系统自然树大招风，避免不了无穷尽的打补丁 纯静态页面，除了被DDOS攻击和HTTP劫持也没什么了 Hexo在本地有完整的备份，结合Git发布，相当于机器猫的任意门了 Hexo
插件和主题 系统本身就是为了插件而设计的，麦咭哥就做了几十个插件，可想其丰富程度了 坦白说不多，也不少，在不跑偏的前提下，总能找到解决方案 面对Wordpress的无所不能，作为静态页面解决方案的Hexo，够用足矣，想要评论功能也是可以有的 Wordpress
